# Задание 1



### Текст задачи



Разработать консольное приложение с дружественным интерфейсом с возможностью выбора заданий для работы с «базой данных (БД)», хранящейся в бинарном файле. Перечень полей (минимум 5), достаточно полно характеризующих заданную в варианте предметную область, предложить самостоятельно (постараться отразить в перечне полей такие, которые требуют разных типов данных). Приложение должно выполнять следующие функции:
1. Чтение базы данных из бинарного файла.
2. Просмотр базы данных.
3. Удаление элементов (по ключу).
4. Добавление элементов.
5. Реализация 4 запросов (формулировки запросов придумать самостоятельно). 2 запроса должны возвращать перечень, 2 запроса одно значение.
В классе должны присутствовать свойства, конструкторы, перегруженный метод ToString(). Весь функционал приложения реализовать в виде методов вспомогательного класса с помощью LINQзапросов.
Предусмотреть обработку возможных ошибок при работе программы.
Пункт 5 оценивается в 2 балла, остальные пункты по 1 баллу. Максимально за лабораторную работу можно получить 6 баллов.



Вариант 10 - Музейные экспонаты.



### Алгоритм решения



1\. Определение класса Exhibit, содержащего поля (\_id, \_name, \_artist, \_year, \_material, \_value), публичные свойства для доступа к ним, три конструктора (по умолчанию, с параметрами, для чтения из BinaryReader), метод Write(BinaryWriter) для сохранения в файл и переопределённый ToString().
2\. Создание вспомогательного класса DatabaseHelper, который инкапсулирует работу с файлом и коллекцией экспонатов:
2.1\. При создании объекта инициализируется путь к файлу и вызывается метод LoadFromFile(), который:
2.1.1\. Проверяет существование файла.
2.1.2. При отсутствии файла создаёт пустой список.
2.1.3. При наличии файла считывает количество записей и построчно создаёт объекты Exhibit с помощью конструктора, принимающего BinaryReader.
2.2. Метод SaveToFile() сохраняет текущий список в бинарный файл.
2.3. Метод ViewAll() выводит на экран все экспонаты, используя ToString().
2.4. Метод AddExhibit(Exhibit) проверяет уникальность ID с помощью цикла, затем добавляет объект в список и вызывает SaveToFile().
2.5. Метод DeleteById(int) ищет экспонат по ID в цикле, при нахождении удаляет и сохраняет изменения.
2.6. Метод GetByMaterial(string) возвращает новый список, в который циклом добавляются экспонаты, материал которых совпадает с заданным.
2.7. Метод GetByYearAfter(int) аналогично возвращает список экспонатов с годом создания больше заданного.
2.8. Метод GetAverageValue() вычисляет сумму стоимостей в цикле и делит на количество экспонатов.
2.9. Метод GetCountByArtist(string) подсчитывает количество экспонатов заданного автора через цикл и счётчик.
2.10.Метод ExistsId(int) проверяет наличие экспоната с указанным ID.

3. В классе Program реализован интерфейс с меню:
3.1. Запуск приложения: создаётся объект DatabaseHelper с именем бинарного файла "exhibits.dat".
3.2. В цикле while (!exit) выводится меню из 8 пунктов (просмотр, добавление, удаление, 4 запроса, выход).
3.3. Для каждого пункта вызывается соответствующий диалоговый метод (AddExhibitDialog(), DeleteExhibitDialog() и т.д.).
3.4. В диалогах используются вспомогательные методы ReadInt, ReadDecimal, ReadNonEmptyString, которые повторяют запрос до получения корректного значения, с проверкой диапазонов и пустых строк.
3.5. При добавлении проверяется уникальность ID; при удалении — существование.
3.6. После успешного добавления или удаления изменения автоматически сохраняются в файл.
